# script.js 能力追加手順書

## 概要

`script.js` の冒頭にある `allAbilities` 配列に1行追加するだけで新しい能力を登録できます。  
追加後に必要な作業は **0〜1箇所のみ**（新グループを作る場合のみ `searchGroups` も編集）。

---

## 1. 各フィールドの意味

```js
{ id:"xxx", group:"グループ名", label:"表示名", cost:100, discount:"10%", value:85, eff:0.8500, searchable:true }
```

| フィールド | 型 | 説明 |
|---|---|---|
| `id` | string | **全能力で一意**の識別子。重複厳禁。命名規則は後述 |
| `group` | string | 能力グループ。UIのアコーディオン区分け・バッジ色に使われる |
| `label` | string | 画面や結果に表示される能力名 |
| `cost` | number \| "変動" | 消費pt。数値固定でない場合は文字列 `"変動"` |
| `discount` | string \| null | 割引率（例: `"10%"`）。なければ `null` |
| `value` | number \| "変動" | 取得できる総合力。数値固定でない場合は `"変動"` |
| `eff` | number | 効率 = `value / cost`（小数。例: 85÷100=0.85）|
| `searchable` | boolean | `true`=計算UI対象、`false`=効率一覧のみ表示 |

---

## 2. `id` の命名規則（推奨）

重複を避けるため、以下のパターンを使ってください。

```
{種別略称}_{コスト}_{割引率の数字}
```

| 能力 | 例 |
|---|---|
| 白 120pt 10%割引 | `w120_10` |
| 金 100pt 割引なし | `g100_0` |
| 白+金 216pt | `wg216` |
| ロックオン系(白) 140pt | `lock_w140` |
| 新能力「○○」100pt 0% | `new_oo_100_0` |

既存の `id` 一覧はファイル先頭の `allAbilities` をCtrl+Fで検索して確認できます。

---

## 3. `eff`（効率）の計算方法

```
eff = value ÷ cost
```

**例**
- 白 108pt → 総合力72 → `72 ÷ 108 = 0.6667`
- 金 96pt → 総合力102 → `102 ÷ 96 = 1.0625`

小数第4位まで記載するのが慣例です（表示は `×100` して `%` 換算されます）。

## 4. 追加パターン別の手順

### パターンA：既存グループへの追加（最も多いケース）

**例）「英雄金+白」に 168pt（割引なし）の行を追加する**

1. `allAbilities` 内の該当コメント（`// 英雄金+白`）を探す
2. 末尾の行の直後に1行追加する

```js
// 英雄金+白
{ id:"hero_gw204",group:"その他",label:"英雄金+白", cost:204, discount:null, value:204, eff:1.0000, searchable:true },
{ id:"hero_gw192",group:"その他",label:"英雄金+白", cost:192, discount:null, value:204, eff:1.0625, searchable:true },
{ id:"hero_gw180",group:"その他",label:"英雄金+白", cost:180, discount:null, value:204, eff:1.1333, searchable:true },
{ id:"hero_gw168",group:"その他",label:"英雄金+白", cost:168, discount:null, value:204, eff:1.2143, searchable:true },  // ← 追加
```

3. **他に作業は不要**。保存してブラウザをリロードすれば反映されます。

---

### パターンB：新しいグループを追加する

**例）「超新規（複合）」という新グループを計算UIに加えたい**

#### ステップ1：`allAbilities` にデータを追加

```js
// 超新規（複合）
{ id:"cho_nc_300", group:"超新規複合", label:"超新規(複合)", cost:300, discount:null, value:250, eff:0.8333, searchable:true },
{ id:"cho_nc_285", group:"超新規複合", label:"超新規(複合)", cost:285, discount:"5%",  value:250, eff:0.8772, searchable:true },
```

#### ステップ2：`searchGroups` 配列に1行追加

`allAbilities` の下にある `searchGroups` を探してください。

```js
const searchGroups = [
  { key:"白",       label:"白",       cls:"gl-白" },
  { key:"金",       label:"金",       cls:"gl-金" },
  { key:"白+金",    label:"白＋金",   cls:"gl-白金" },
  { key:"ひらめき", label:"ひらめき", cls:"gl-ひらめき" },
  { key:"その他",   label:"その他",   cls:"gl-その他" },
  { key:"超新規複合", label:"超新規(複合)", cls:"gl-超新規" },  // ← 追加
];
```

- `key` は `allAbilities` の `group` と**完全一致**させてください
- `cls` は `style.css` のクラス名（既存のものを流用可）

#### ステップ3（任意）：バッジ色を新設したい場合

`style.css` に追記します（既存クラス流用なら不要）：

```css
.gl-新グループ { color: #ff99cc; background: rgba(255,153,204,0.08); }
```

`groupBadgeClass` オブジェクト（`script.js` 内）にも追記します：

```js
const groupBadgeClass = {
  ...
  '超新規複合': 'tb-超新規',  // ← 追加（既存クラス流用の例）
};
```

---

### パターンC：効率一覧にのみ表示（計算対象外）

`searchable: false` にします。UIのアコーディオンには表示されず、効率一覧モーダルにのみ出現します。

```js
{ id:"new_boss_10", group:"ノラモン・ボスモン", label:"ノラモン・ボスモン", cost:"変動", discount:"10%", value:"変動", eff:1.1111, searchable:false },
```

---

### パターンD：金+白のようなセット能力（リンク機能対応）

「ロックオン系(金+白)」と「ロックオン系(白)」のように、セット能力と部分能力を**両方登録する**ケースです。

1. 両方を `allAbilities` に追加する（パターンAと同様）
2. ユーザーがツール上で個数を設定した後、🔗リンクボタンで紐づければ計算時に排他制御されます

データ追加側での特別な作業は不要です。リンクはユーザー操作で行います。

---

## 5. よくあるミスと確認方法

| ミス | 症状 | 対処 |
|---|---|---|
| `id` が重複 | 一方の能力が無視される | ファイル内でidをCtrl+F検索して重複確認 |
| `group` のスペルミス | アコーディオンに表示されない | `searchGroups` の `key` と完全一致しているか確認 |
| `eff` の計算ミス | 効率一覧の順位がおかしい | `value ÷ cost` を電卓で再確認 |
| `searchable` の指定漏れ | 計算対象に含まれない | `true` / `false` を明示的に記載 |
| `discount:null` を書き忘れ | エラーまたは表示崩れ | 割引なしでも必ず `discount:null` を書く |

## 6. 追加後のチェックリスト

- [ ] ブラウザをハードリロード（Ctrl+Shift+R）して画面に能力が表示されるか確認
- [ ] 効率一覧モーダルを開き、正しい順位に表示されているか確認
- [ ] 計算ボタンを押して結果に新能力が使われるか確認
- [ ] セット能力を追加した場合は🔗リンクボタンで排他制御が機能するか確認
